## Data Storytelling - Framework

### Estructura Narrativa

```
┌─────────────────────────────────────────────────────────────────┐
│                    ESTRUCTURA DE LA HISTORIA                    │
├─────────────────┬─────────────────┬─────────────────────────────┤
│    CONTEXTO     │    CONFLICTO    │        RESOLUCION           │
│   (Setup)       │   (Tension)     │       (Payoff)              │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ • Situacion     │ • Problema      │ • Insights                  │
│   actual        │   identificado  │ • Recomendaciones           │
│ • Por que       │ • Impacto en    │ • Acciones                  │
│   importa       │   el negocio    │   sugeridas                 │
│ • Audiencia     │ • Urgencia      │ • Siguiente paso            │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

### Los 3 Elementos del Data Storytelling

```
                         ┌─────────────┐
                         │   DATOS     │
                         │  (Hechos)   │
                         └──────┬──────┘
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                 │
              ▼                 ▼                 ▼
      ┌───────────────┐ ┌───────────────┐ ┌───────────────┐
      │  VISUALES     │ │  NARRATIVA    │ │   AUDIENCIA   │
      │  (Graficos)   │ │  (Historia)   │ │  (Contexto)   │
      └───────────────┘ └───────────────┘ └───────────────┘
```

## Guia de Presentacion (10-15 minutos)

### Minuto 0-2: Contexto
- Presentacion del equipo
- Descripcion de la PYME
- Problema de negocio a resolver

### Minuto 2-5: El Dashboard
- Walkthrough de la interfaz
- Explicacion de KPIs principales
- Demostracion de interactividad

### Minuto 5-10: Insights
- Hallazgos clave del analisis
- Visualizaciones que soportan los insights
- Conexion con Data Mining (Fase 2)

### Minuto 10-13: Recomendaciones
- Acciones sugeridas basadas en datos
- Impacto esperado
- Proximos pasos

### Minuto 13-15: Q&A
- Responder preguntas de "stakeholders"
- Demostrar conocimiento profundo

## Tips para Presentacion Efectiva

### DO's
- [ ] Conocer la audiencia (simular CFO/CTO)
- [ ] Comenzar con el "so what?" (por que importa)
- [ ] Usar visualizaciones para soportar, no dominar
- [ ] Pausar para enfatizar puntos clave
- [ ] Hacer contacto visual
- [ ] Practicar el timing

### DON'Ts
- [ ] Leer slides/dashboard textualmente
- [ ] Abrumar con demasiados numeros
- [ ] Ignorar preguntas dificiles
- [ ] Exceder el tiempo asignado
- [ ] Usar jerga tecnica sin explicar
- [ ] Mostrar nerviosismo excesivo

## Rubrica Proyecto Fase 3 (20%)

### Dashboard (12 puntos)

| Criterio | Excelente (4) | Bueno (3) | Regular (2) | Deficiente (1) |
|----------|---------------|-----------|-------------|----------------|
| Calidad tecnica | Sin errores, optimizado | Funcional | Algunos errores | No funciona |
| Diseno visual | Principios Few/Tufte aplicados | Adecuado | Chartjunk presente | Confuso |
| Interactividad | Drill-down, slicers, filtros | Parcial | Basica | Ausente |

### Presentacion Oral (8 puntos)

| Criterio | Excelente (4) | Bueno (3) | Regular (2) | Deficiente (1) |
|----------|---------------|-----------|-------------|----------------|
| Narrativa | Historia clara y convincente | Coherente | Fragmentada | Confusa |
| Dominio tema | Responde todo con profundidad | Adecuado | Superficial | Desconoce |
